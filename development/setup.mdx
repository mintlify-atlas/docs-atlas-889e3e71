## System Requirements

Before you begin, make sure you have the following installed:

| Tool | Minimum Version | Notes |
|------|----------------|-------|
| Node.js | 18.x or later | LTS recommended |
| npm | 9.x or later | Ships with Node |
| Git | Any recent version | For cloning the repo |

## Clone and Install

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone <repository-url>
    cd polygon-animals
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```

    This installs all production and development dependencies defined in `package.json`.
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    Vite starts a local dev server with Hot Module Replacement (HMR). Open the URL printed in your terminal (typically `http://localhost:5173`).
  </Step>
</Steps>

## Available Scripts

All scripts are defined in `package.json`:

| Script | Command | Description |
|--------|---------|-------------|
| `dev` | `vite` | Starts the Vite dev server with HMR |
| `build` | `vite build` | Builds the app for production into `dist/` |
| `preview` | `vite preview` | Serves the production build locally for verification |
| `lint` | `eslint .` | Runs ESLint across all `.js` and `.jsx` files |

## Vite Configuration

The project uses a minimal Vite configuration in `vite.config.js`:

```js
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
  plugins: [react()],
  server: {
    allowedHosts: true,
    host: true,
  },
});
```

**Key settings:**

- **`plugins: [react()]`** — Enables JSX transform, Fast Refresh, and React-specific Vite optimisations via `@vitejs/plugin-react`.
- **`host: true`** — Binds the dev server to `0.0.0.0`, making it accessible on your local network (useful for testing on mobile devices).
- **`allowedHosts: true`** — Allows connections from any hostname, which is helpful when running inside containers or cloud dev environments.

<Note>
  The `hmr.clientPort` option is commented out in the config. Uncomment and set it to `443` only if you need HMR to work through an HTTPS reverse proxy.
</Note>

## ESLint Setup

ESLint is configured in `eslint.config.js` using the flat config format introduced in ESLint 9:

```js
import js from '@eslint/js'
import globals from 'globals'
import reactHooks from 'eslint-plugin-react-hooks'
import reactRefresh from 'eslint-plugin-react-refresh'
import { defineConfig, globalIgnores } from 'eslint/config'

export default defineConfig([
  globalIgnores(['dist']),
  {
    files: ['**/*.{js,jsx}'],
    extends: [
      js.configs.recommended,
      reactHooks.configs.flat.recommended,
      reactRefresh.configs.vite,
    ],
    languageOptions: {
      ecmaVersion: 2020,
      globals: globals.browser,
      parserOptions: {
        ecmaVersion: 'latest',
        ecmaFeatures: { jsx: true },
        sourceType: 'module',
      },
    },
    rules: {
      'no-unused-vars': ['error', { varsIgnorePattern: '^[A-Z_]' }],
    },
  },
])
```

**What this enforces:**

- **`js.configs.recommended`** — Standard ESLint rules for common JS errors.
- **`reactHooks.configs.flat.recommended`** — Enforces the Rules of Hooks (e.g. no calling hooks inside conditions).
- **`reactRefresh.configs.vite`** — Warns if a module exports something that would break Fast Refresh.
- **`no-unused-vars`** — Errors on unused variables, but ignores identifiers that start with a capital letter or underscore (e.g. component names imported for JSX use).

Run the linter with:

```bash
npm run lint
```
