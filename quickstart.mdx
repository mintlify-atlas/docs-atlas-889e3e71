Polygon Animals is a React + Vite application. You need Node.js and Git installed before you begin.

## Prerequisites

| Requirement | Version |
|-------------|----------|
| Node.js | 18 or later |
| npm | 9 or later (bundled with Node.js) |
| Git | any recent version |

<Tip>
Run `node -v` and `npm -v` in your terminal to confirm your versions before continuing.
</Tip>

## Setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone <your-repository-url>
    cd polygon-animals
    ```
  </Step>

  <Step title="Install dependencies">
    From the project root, install all required packages:

    ```bash
    npm install
    ```

    This installs the following key packages:

    | Package | Version | Purpose |
    |---------|---------|----------|
    | `react` | ^19.2.0 | UI framework |
    | `react-dom` | ^19.2.0 | DOM rendering |
    | `react-router-dom` | ^7.12.0 | Client-side routing |
    | `framer-motion` | ^12.29.0 | Scroll animations and parallax |
    | `@react-three/fiber` | ^9.5.0 | React renderer for Three.js |
    | `@react-three/drei` | ^10.7.7 | Three.js helpers and abstractions |
    | `three` | ^0.182.0 | 3D graphics engine |
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    Vite starts a local server at `http://localhost:5173` (or the next available port). The terminal output confirms the exact URL.

    <Note>
    The first page you see is `/stages` — the root path `/` immediately redirects there via React Router's `<Navigate>` component.
    </Note>
  </Step>
</Steps>

## Available scripts

| Script | Command | Description |
|--------|---------|-------------|
| Dev server | `npm run dev` | Starts Vite dev server with hot module replacement |
| Production build | `npm run build` | Compiles to `dist/` for deployment |
| Preview build | `npm run preview` | Serves the production build locally |
| Lint | `npm run lint` | Runs ESLint across the project |

## Route overview

Once the dev server is running, you can navigate to any of these routes directly in the browser:

| Route | Page | Description |
|-------|------|-------------|
| `/` | — | Redirects to `/stages` |
| `/stages` | Stages | Entry point — choose a section to explore |
| `/landing` | Landing | Scrollable showcase of all 10 animals |
| `/detail?id=<animalId>` | Detail | Polygon animation and conservation info for one animal |
| `/polygon` | Polygon | Horizontal parallax storytelling about low-poly art |
| `/order` | Orderer | Internal tool for assigning shard animation order |

## What to expect

When you open the app at `/stages`, a two-screen animated intro sequence plays on first visit (`Intro1` → `Intro2`). After completing the intro — or on subsequent visits — you land on the stage selection screen with four vertically split panels.

From there:

- Click **stage 1** (สำรวจท้องทะเล) to enter the scrolling `/landing` page
- Click **stage 2** (รายละเอียด) to open the detail viewer at `/detail`
- Click **stage 4** (ทำไมต้อง Polygon) to open the horizontal parallax experience at `/polygon`

<Info>
Stage 3 (ภัยคุกคาม) does not have a route assigned in the current version of the app.
</Info>

## Vite dev server notes

- **Hot module replacement (HMR)** — React components and CSS update instantly without a full page reload.
- **Assets** — Images, audio files, and SVGs in `src/assets/` are served by Vite and imported directly with ES module syntax.
- **Port** — Vite defaults to `5173`. If that port is in use, it automatically increments to `5174`, and so on.
- **Build output** — `npm run build` writes a production bundle to `dist/`. The app is a standard SPA; configure your hosting provider to serve `index.html` for all routes.
